## 1. Purpose

This policy establishes governance for managing user access throughout the employment lifecycle to ensure appropriate access rights are granted, modified, and revoked in a timely and secure manner.

---

## 2. Scope

This policy applies to:
- All new joiners (employees, contractors, consultants)
- All movers (role changes, transfers, promotions)
- All leavers (terminations, resignations, contract ends)
- All user accounts and access rights
- All information systems and resources

---

## 3. Governance

### Policy Owner
The Information Security Manager owns this policy in collaboration with HR/function leads.

### Key Stakeholders
| Role | Responsibility |
|------|---------------|
| **HR/Function Lead** | Notify IT of JML events, verify identity |
| **Information Security Manager** | Approve access requests, review exceptions |
| **System Administrators** | Implement access changes, maintain records |
| **Line Managers** | Request access, confirm business need, notify of departures |

---

## 4. Joiners (New Users)

### 4.1 Pre-Employment
**Before start date:**
- [ ] Identity verification completed
- [ ] Background checks performed (if applicable)
- [ ] Confidentiality agreement signed
- [ ] Security awareness briefing scheduled
- [ ] Access request submitted by line manager
- [ ] Risk assessment for elevated privileges (if applicable)

### 4.2 Access Provisioning
**On start date:**
- [ ] User account created with default/minimal access
- [ ] Role-based access granted per job requirements
- [ ] Multi-factor authentication enrolled
- [ ] Acceptable use policy acknowledged
- [ ] Security awareness training completed
- [ ] Asset allocation (laptop, tokens, etc.)

**Principles:**
- **Least Privilege:** Access granted on need-to-know basis
- **Role-Based:** Access aligned to job function
- **Timely:** Access available on first working day

### 4.3 Documentation
- Access request form retained
- Approval records maintained
- Asset register updated
- Training completion recorded

---

## 5. Movers (Role Changes)

### 5.1 Change Triggers
- Promotion or demotion
- Department transfer
- Project assignment changes
- Temporary role changes
- Additional responsibilities

### 5.2 Access Review Process
**When role changes:**
- [ ] Line manager submits access modification request
- [ ] Current access rights reviewed
- [ ] New role requirements assessed
- [ ] Unnecessary access identified for removal
- [ ] New access approved by system owner
- [ ] Changes implemented within 24 hours
- [ ] User informed of changes

### 5.3 Access Transfer Principles
- **Review First:** Audit existing access before changes
- **Remove Unnecessary:** Revoke access no longer required
- **Add Required:** Grant access needed for new role
- **Document:** Maintain audit trail of all changes
- **Timely:** Complete within 24 hours of notification

---

## 6. Leavers (Departures)

### 6.1 Types of Departure
- Resignation
- Termination
- Contract end
- Retirement
- Sabbatical (temporary)

### 6.2 Notification Requirements
| Departure Type | Notice Period | IT Notification Required |
|----------------|---------------|-------------------------|
| Resignation | Per contract | Immediate upon acceptance |
| Termination | Immediate | Immediate |
| Contract End | Per contract | 5 working days before |
| Retirement | Per contract | 10 working days before |

### 6.3 Offboarding Checklist
**Before departure date:**
- [ ] Leaver notification received from HR/Manager
- [ ] Return of assets scheduled (laptop, tokens, documents)
- [ ] Data transfer plan agreed (if applicable)
- [ ] Exit interview scheduled

**On departure date:**
- [ ] All system access revoked
- [ ] Remote access disabled
- [ ] VPN access removed
- [ ] Email forwarding configured (if required)
- [ ] Shared account passwords changed
- [ ] Assets returned and checked

**Within 24 hours of departure:**
- [ ] User account disabled or deleted
- [ ] Access rights removed from all systems
- [ ] Data access reviewed for anomalies
- [ ] Final access audit completed

**Within 7 days:**
- [ ] Data retention requirements confirmed
- [ ] User data archived or deleted per policy
- [ ] Exit interview completed
- [ ] Final access review documented

### 6.4 Emergency Revocation
If immediate termination is required:
- All access revoked within 1 hour
- Physical access disabled immediately
- Remote sessions terminated
- Line manager and security notified

---

## 7. Access Reviews

### 7.1 Regular Reviews
- **Quarterly:** Privileged access review
- **Annually:** Full access review for all users
- **Upon event:** Post-mover review within 30 days

### 7.2 Review Process
- HR provides current staff list
- System administrators extract access rights
- Line managers verify business need
- Unnecessary access identified and removed
- Results documented

---

## 8. Exceptions and Escalation

### 8.1 Temporary Access
- Must be time-bound with expiry date
- Requires additional approval
- Automatically revoked if not renewed
- Documented with business justification

### 8.2 Emergency Access
- Break-glass procedures documented separately
- Post-event review required
- Access logged and audited

### 8.3 Escalation Path
1. System Administrator (implementation)
2. Information Security Manager (approval, exceptions)
3. Senior Management (policy conflicts, high-risk decisions)

---

## 9. Audit and Compliance

### 9.1 Records Maintained
- All access requests and approvals
- All JML notifications and actions
- Access review results
- Exception documentation

### 9.2 Audit Trail
All JML activities logged with:
- Date and time
- Action taken
- Person performing action
- Authorisation reference

### 9.3 Compliance Monitoring
- Quarterly compliance reports
- Annual policy effectiveness review
- Incident trend analysis
